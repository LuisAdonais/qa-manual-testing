# AI Output Validation

---

## Purpose

In this file, I define how I validate the test artifacts generated by AI before they are used in the testing process.

---

## Relation to the AI Usage Flow

This file is part of the AI flow:

- Test Basis
- ChatGPT (generation)
- Evaluation Prompt (validation)
- Cursor (format and review)
- Final manual validation

I keep the project in ChatGPT to ensure consistency in prompts and context.

Validation is mandatory before using test artifacts.

---

## Evaluation Context

To validate the test artifacts, I use a structured prompt:

- [Evaluation Prompt](../prompts/evaluation-prompt.md)

This prompt defines the role of QA reviewer who:

- Evaluates the quality of the artifact
- Does not generate or modify content
- Does not assume missing information
- Detects inconsistencies and problems

The evaluation is based on:

- ISTQB terminology
- Separation by phases of the STLC
- Traceability between artifacts
- Validation against the test basis
- Coverage and clarity
- AI risks (hallucinations, reasoning errors, biases)
- Output quality (accuracy, relevance, consistency, completeness)

---

## Input (Test Basis)

To validate the test artifacts, I use:

- Requirements
- User Stories
- Acceptance Criteria

These elements are the test basis.

---

## AI Risks

During validation I consider these risks:

- Hallucinations: artifacts not based on the test basis
- Reasoning errors: incorrect logic in the Test Case
- Inconsistencies: missing relationship between Requirement, Test Condition, and Test Case
- Formatting problems: inconsistent or unclear Markdown structure

---

## How I Validate

I validate the test artifacts in two steps:

### 1. AI-Assisted Validation

- I use the evaluation prompt
- I identify problems in:
  - Terminology
  - Structure
  - Traceability
  - Clarity

### 2. Manual Validation

- I compare the artifacts against the test basis
- I confirm that:
  - The Test Case comes from a Requirement
  - The Test Case covers a Test Condition
  - The expected result is clear and verifiable
  - The steps are executable
  - The format is clear and consistent

---

## Validation Checklist

- Is it based on the test basis?
- Does it cover the Test Condition?
- Is the expected result verifiable?
- Are the steps clear and executable?
- Is the format consistent and readable?

---

## Validation Result

- Approved: the artifact can be used in the Test Execution
- Needs correction: I adjust the artifact or refine the prompt

---

## Notes

> [!NOTE]
> I do not use AI-generated artifacts without validation.
> I keep the project context in ChatGPT for consistency.
> The evaluation prompt helps to detect problems, but the final validation is the responsibility of the tester.
